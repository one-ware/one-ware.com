## Datasets
### Dataset 1: Chips
[This dataset](https://www.kaggle.com/datasets/concaption/pepsico-lab-potato-quality-control) contains quality control images from a food processing plant that produces potato chips. During the production process, the chips can get burned making them unsuitable for consumption. The goal is to create an AI that can classify an image of a potato chip as defective or non-defective.

### Dataset 2: Strawberries
This dataset contains images of a conveyor belt that is transporting candy strawberries but can also contain foreign objects. The goal is to create an AI that can detect the objects on the conveyor belt to sort out any foreign objects. The dataset was created artificially by placing images of objects on top of images of the conveyor belt.

### Dataset 3: Coffee Mugs
This dataset contains images of our ONE WARE coffee mugs. We created artificial defects by drawing scratches onto the images of the mugs. The goal is to create an AI model that can detect these scratches for quality control.

### Dataset 4: GC10-Det Subset
The [GC10-Det dataset](https://universe.roboflow.com/datasets-aaoen/gc10-det-hu31n) contains images from a real-world industrial setting and is used for metallic surface defect detection. The original dataset contains ten different types of defects, but for this exercise we only use a single type of defect to keep the training time short.

## Your task

0. **Set up OneWare Studio**: Follow the instructions in our [setup guide](/docs/one-ai/getting-started/installation) to install OneWare Studio, the OneAI extension and to create an account. You can use the code ``webinar-2025-12-18`` in the [cloud](https://cloud.one-ware.com/credits) to get free credits for this workshop.
1. **Choose one of the exercise datasets and download it**
2. **Create a new project** by clicking on ``AI`` > ``OneAI Project Generator``
3. **Import the exercise dataset**. You can verify whether the import was successful by clicking on one of the images and checking the annotations.
4. **Set up filters**: The default settings will already create a good starting point, but we can improve the model by tuning the filters to our application.
5. **Configure the model settings**: provide additional information about the application to obtain the best possible model.
6. **Train the model** for the specified time.
7. **Test the model** and look for cases where the performance can be improved.
